📖 Pivot Reference Guide

Controls explained in plain English + how the voice pipeline works.

🧠

LLM Intelligence

Human Response Bounds

Controls how much the AI says in each turn. Normal settings apply to quick back-and-forth. Detail settings kick in automatically when you ask for an explanation or say "tell me more".

Control What it does & recommended range
Normal Tokens
50 – 1000		 Max number of words (tokens) in a normal reply. 100–200 keeps answers short and conversational — ideal for a real-time voice agent. Higher values allow longer answers but increase latency.
Normal Sentences
1 – 10		 Max sentences the agent speaks before pausing. 2–3 feels like natural speech. More sentences = longer monologues. The agent streams each sentence to TTS as it's generated — this controls how many it will generate before stopping.
Detail Tokens
100 – 2000		 Token cap used when you ask for a detailed explanation. 500–800 gives thorough answers without being too long. Triggered automatically by keywords like "explain", "tell me more", "how does".
Detail Sentences
1 – 15		 Sentence cap for detail mode. 5–8 gives a thorough but not exhausting answer. Works together with Detail Tokens — whichever limit is hit first stops the reply.

Voice & Audio Synthesis

Human Cadence

Controls how the agent's voice sounds in your browser. These are applied to the raw PCM audio chunks delivered by Cartesia TTS before playback.

Control What it does & recommended range
Speech Rate
0.5× – 2.0×		 How fast the agent talks. 0.9×–1.1× sounds natural. Below 0.8× is noticeably slow; above 1.3× can feel rushed. Applied client-side via AudioBuffer playback rate — no extra latency.
Volume
0% – 100%		 Output volume of the agent's voice. Does not affect your microphone. 100% is default; reduce if you're using headphones and the agent is too loud.
TTS Model Which Cartesia voice model synthesises the audio:
• Sonic English — best quality, lowest latency for English (default)
• Sonic Multilingual — supports mixed language / code-switching
• Cartesia Neural — standard neural voice, slightly higher latency

STT & VAD Sensitivity

Pause Allowance

Controls how the browser detects when you speak and when you've finished. VAD = Voice Activity Detection.

Control What it does & recommended range
STT Language The language passed to the browser's Web Speech API for recognition. English (US) gives the best accuracy. Set this to match the language you will speak — mixing languages requires Sonic Multilingual TTS model.
VAD Silence Timeout
200ms – 1500ms		 How long the system waits after you stop speaking before it considers your utterance complete and sends it to the LLM. 400–500ms feels natural. Too short (200ms) and mid-sentence pauses trigger early sends. Too long (1000ms+) makes the agent feel sluggish.
Live Mic Energy Visual indicator only — shows your real-time microphone energy level. Useful for diagnosing mic input issues. No configuration needed.

Human Interruption & Delays

Natural Cutoff

Controls what happens the instant you interrupt the agent mid-speech. These settings shape how "human" the cutoff feels — whether the agent stops abruptly or eases out gracefully.

Control What it does & recommended range
Barge-in Cutoff Delay
0ms – 500ms		 After VAD detects your voice, the system waits this long before cancelling the agent's speech. 0ms = instant stop (most responsive). Higher values (100–200ms) give the agent a small grace period to finish the current word — can sound more natural but feels slower. For production use, keep at 0–50ms.
Audio Chunk Buffer
0ms – 200ms		 A small buffer added to each audio chunk before scheduling it for playback. Helps smooth out gaps if the network has variable latency. 20–40ms is the sweet spot. Too high adds noticeable delay. Too low risks audio glitches on poor connections.
Interruption Cutoff Mode How the agent's voice stops when you interrupt:
• Soft Audio Fade (50ms) — the agent's voice fades out in 50ms. Sounds natural and human. (Recommended)
• Instant Silence (0ms) — the audio stops dead immediately. More robotic but guarantees zero bleed-over.

Session Controls Panel (⚙�? button → side drawer)

Live controls you can hit during a session. Open the side drawer with the ⚙�? Session Controls Panel button in the header.

🔇 Mute / 🎤 Unmute Mic
Silences your microphone without disconnecting. The agent will not detect your voice until you unmute.
�?� Stop / ▶ Start STT
Pauses the speech recogniser entirely. Useful if you're in a noisy environment and want to stop false activations.
🛑 Stop / Interrupt Turn
Immediately cancels the agent's current reply — same as interrupting with your voice, but via button. Useful in testing or when mic VAD isn't triggering.
🔄 Reset Session
Clears the conversation history for the current session. The agent forgets what was said. Useful to start a fresh topic without reconnecting.
🔗 Reconnect
Re-establishes the WebRTC / WebSocket connection if it dropped without refreshing the page.
⚡ Shutdown API
Sends a shutdown signal to the backend server process. Use only intentionally — this ends all active sessions.

🔗 The Voice Pipeline (What Happens When You Speak)

Every time you say something, Pivot runs your words through a chain of workers in sequence:

🎙�?
STT Worker
Turns spoken words into text
🧠
FSM Worker
Decides what to do next (new reply, interrupt, etc.)
💬
LLM Worker
Streams reply sentence-by-sentence
🔊
TTS Worker
Converts each sentence to audio
📡
Playback Worker
Streams audio chunks to browser
⚡
Interrupt Monitor
Cancels current turn instantly when you speak

📊 Telemetry — Why Every Event Is Tracked

Every turn emits the same 11 events in order — whether normal, interrupted, or errored:

VAD start → STT end → LLM request → LLM first token → LLM complete
→ TTS start → TTS chunk(s) → TTS complete → Playback start → Playback end → Turn complete

If a stage is cancelled, it records null + a cancellation_reason so the waterfall chart is always complete.

⚡ How Interruption Works

  1. You speak → browser VAD fires instantly.
  2. A cancel signal is sent to the server.
  3. Server sets a per-session cancel flag — LLM + TTS workers stop immediately.
  4. Browser increments a playback generation counter — buffered audio is discarded.
  5. The turn is logged with a cancellation_reason.
